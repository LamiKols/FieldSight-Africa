API Documentation

Safe developer onboarding for `/api/v1/document-metadata`.

Authentication

Use an active API key linked to an active API client. Keys are verified by prefix plus SHA-256 hash; raw secrets are not stored or displayed after creation.

Authorization: Bearer <your_api_secret>
X-API-Key: <your_api_secret>

The key or client must include the `document_metadata:read` scope.

Endpoint

GET /api/v1/document-metadata

Optional filters: `document_id`, `document_type`, `crop`, and `region`.

curl -H "Authorization: Bearer <your_api_secret>" \
  "{{ request.url_root.rstrip('/') }}/api/v1/document-metadata?region=SW&crop=Ginger"

Access Limits And Gates

API access is commercial-review based. Submitting an enquiry does not create an API client, key, entitlement, or scope.

Every metadata row must pass Phase 3.4 gates: publish control, consent, admin approval, verification, expiry, redaction, extraction/reconciliation, high-risk checks, and entitlement scope.

Usage is logged in `ApiUsageEvent`. Allowed and blocked document-level attempts are logged through `DocumentAccessLog`.

Never Returned

The API does not return document files, private storage paths, original filenames, stored filenames, file hashes, actor names, contact details, raw extraction text, evidence excerpts, or partner-sensitive notes.
